01 · El Desafío: Fricción Operativa

El ERP que nadie
usa en campo.

Los ERPs modernos son potentes, pero presentan barreras críticas para el usuario de campo: demasiados clics, sin acceso móvil real, y resistencia al cambio.

  • Complejidad de interfaz: +12 campos obligatorios en CRM, navegación de 4 pantallas.
  • Baja movilidad: comerciales registran de memoria al volver a oficina, perdiendo contexto.
  • Resistencia al cambio: el personal prefiere herramientas familiares como Telegram o WhatsApp.

02 · La Solución

Habla.
El resto lo hace el BOT.

El bot de Telegram captura notas de voz en lenguaje natural, las transcribe con Gemini 2.5 Flash y escribe directamente en Odoo ERP.

  • Cero curva de aprendizaje — Telegram ya lo usan todos los comerciales.
  • Modelo-agnóstico — CRM, contactos, stock, ventas, facturas, tareas.
  • Human in the Loop — confirmación obligatoria antes de escribir en BD.
  • Desplegado en Google Cloud Run Always On — latencia mínima garantizada.

async def extract_structured_data(self, text, model_name): # 1. Obtener el esquema REAL del modelo desde Odoo schema = await self.odoo.get_model_fields(model_name) # 2. Construir mapa enriquecido: tipo, etiqueta, relaciones rich_schema = {} for k, v in schema.items(): field_desc = f"{v['type']} - {v['string']}" if v['type'] == 'many2one': field_desc += f" | Related: {v['relation']}" rich_schema[k] = field_desc # 3. Inyectar esquema dinámico en el prompt de Gemini prompt = f"""Extract data for Odoo model '{model_name}'. Schema: {json.dumps(rich_schema)[:3500]} Rules: Return JSON compatible with Odoo.""" res = await client.generate_content(model=MODEL, contents=prompt) return json.loads(res.text.strip())
ai_processor.py — understand_intent()
async def understand_intent(self, text, last_context): """ Determina acción + modelo Odoo desde lenguaje natural """ # Memoria de sesión: entiende "ese cliente", "el de antes" if last_context: context_hint = f"Last Record: {last_context}" prompt = f""" Analyze: "{text}" 1. Action: create | update | search | analyze | execute 2. Model: crm.lead, res.partner, stock.quant... 3. Execute: action_confirm, button_validate, action_set_won, action_set_lost... Return JSON: {"action":"...", "model":"..."} """ res = await client.generate_content(model=MODEL, contents=prompt) return json.loads(res.text)

CPU solo durante procesamiento. Cold Start activo. Peor caso de producción.

2 vCPU · instancias mínimas=1. Configuración de producción real recomendada.
